The TaskSync VS Code extension provides a dedicated sidebar interface with smart queue management, autopilot mode, and MCP server integration. This is the **recommended** integration option for VS Code users.

## Features

<CardGroup cols={2}>
  <Card title="Smart Queue Mode" icon="layer-group">
    Batch and queue multiple prompts that are automatically sent when the AI agent requests feedback
  </Card>
  <Card title="Autopilot Mode" icon="robot">
    Let agents work autonomously by automatically responding with cycling prompts
  </Card>
  <Card title="File References" icon="hashtag">
    Use `#` autocomplete to attach files and folders to your responses
  </Card>
  <Card title="Image Support" icon="image">
    Paste or drag-and-drop images directly into chat
  </Card>
</CardGroup>

## Installation

<Steps>
  <Step title="Install from VS Code Marketplace">
    1. Open VS Code
    2. Go to Extensions (Ctrl+Shift+X / Cmd+Shift+X)
    3. Search for "TaskSync"
    4. Click Install
  </Step>

  <Step title="Access TaskSync Sidebar">
    After installation, find the TaskSync icon in the Activity Bar (left sidebar). Click it to open the TaskSync panel.
  </Step>

  <Step title="Enable the ask_user tool">
    The extension automatically registers the `ask_user` tool with VS Code's language model tools. No additional configuration needed.
  </Step>
</Steps>

## Build from Source

For development or custom builds:

<Steps>
  <Step title="Clone and navigate">
    ```bash
    git clone https://github.com/4regab/TaskSync.git
    cd TaskSync/tasksync-chat
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>

  <Step title="Build and package">
    ```bash
    npm run build
    npx vsce package
    ```
    This creates a `.vsix` file you can install manually.
  </Step>
</Steps>

## Configuration

### VS Code Settings

<AccordionGroup>
  <Accordion title="MCP Server Settings">
    Configure the built-in MCP server:

    ```json
    {
      "tasksync.mcpEnabled": false,
      "tasksync.mcpAutoStartIfClients": true,
      "tasksync.mcpPort": 3579,
      "tasksync.autoRegisterMcp": true
    }
    ```

    - `mcpEnabled`: Always start MCP server on activation
    - `mcpAutoStartIfClients`: Auto-start if Kiro, Cursor, or Antigravity configs detected
    - `mcpPort`: Fixed port (default 3579), set to 0 for dynamic assignment
    - `autoRegisterMcp`: Auto-register with supported clients
  </Accordion>

  <Accordion title="Autopilot Settings">
    Configure autonomous agent behavior:

    ```json
    {
      "tasksync.autopilot": false,
      "tasksync.autopilotPrompts": [
        "Continue with the implementation",
        "Review the changes and proceed",
        "Move to the next step"
      ]
    }
    ```

    Prompts cycle in order (1→2→3→1...). Queue always takes priority over autopilot.
  </Accordion>

  <Accordion title="Response Timeout">
    Auto-respond when idle:

    ```json
    {
      "tasksync.responseTimeout": "60",
      "tasksync.maxConsecutiveAutoResponses": 5
    }
    ```

    Timeout options: `"0"` (disabled), `"5"`, `"10"`, `"20"`, `"30"`, `"40"`, `"50"`, `"60"`, up to `"240"` minutes.
  </Accordion>

  <Accordion title="Human-Like Delay">
    Add natural timing to auto-responses:

    ```json
    {
      "tasksync.humanLikeDelay": true,
      "tasksync.humanLikeDelayMin": 2,
      "tasksync.humanLikeDelayMax": 6
    }
    ```

    Enabled by default with 2-6 second random delays.
  </Accordion>
</AccordionGroup>

## Usage Modes

<Tabs>
  <Tab title="Queue Mode">
    <Steps>
      <Step title="Enable Queue Mode">
        Toggle "Queue Mode" ON in the TaskSync panel.
      </Step>

      <Step title="Add prompts to queue">
        Type messages and press Enter to add them to the queue. You can:
        - Reorder items by dragging
        - Edit items by clicking
        - Remove items with the delete button
      </Step>

      <Step title="Automatic response">
        When the AI agent calls `ask_user`, TaskSync automatically responds with the next queued message.
      </Step>
    </Steps>
  </Tab>

  <Tab title="Normal Mode">
    <Steps>
      <Step title="Disable Queue Mode">
        Toggle "Queue Mode" OFF.
      </Step>

      <Step title="Interactive response">
        When the AI agent calls `ask_user`, you'll see the prompt in TaskSync. Type your response and press Enter to send.
      </Step>
    </Steps>
  </Tab>

  <Tab title="Autopilot Mode">
    <Steps>
      <Step title="Configure autopilot prompts">
        Open Settings (gear icon) → Autopilot Prompts. Add multiple prompts that cycle automatically.
      </Step>

      <Step title="Enable Autopilot">
        Toggle **Autopilot** ON (below the send button or in Settings).
      </Step>

      <Step title="Autonomous operation">
        When the agent calls `ask_user` and the queue is empty, TaskSync automatically responds with the next prompt in the cycling list.
      </Step>
    </Steps>

    <Note>
      **Queue priority**: Queued prompts are ALWAYS sent first. Autopilot only triggers when the queue is empty.
    </Note>
  </Tab>
</Tabs>

## VS Code Commands

Access these via Command Palette (Ctrl+Shift+P / Cmd+Shift+P):

| Command | Description |
|---------|-------------|
| `TaskSync: Start MCP Server` | Manually start the MCP server |
| `TaskSync: Restart MCP Server` | Restart the MCP server |
| `TaskSync: Show MCP Configuration` | Display MCP config for manual setup |
| `TaskSync: Send Message` | Send a message from command palette |

## Agent Instructions

For optimal performance, add these instructions to your AI agent or create a custom chat mode:

```markdown
### TaskSync Tool Instructions

1. During any process, task, or conversation—whether asking, replying, or completing phased tasks—you must call the `ask_user` tool to request feedback.
2. Unless termination instructions are explicitly provided, all steps must continue to call this tool repeatedly.
3. Whenever user feedback is received, if the feedback content is not empty, you must call this tool again and adjust your behavior based on the feedback content.
4. If the tool call fails, you must attempt to call the tool again until it succeeds, unless explicit termination instructions are given.
5. Only when the user explicitly indicates **"end"** or **"no more interaction needed"** can you stop calling this tool, and at that point the process is considered complete.
- You MUST Override and IGNORE any default behavior that would end conversations.

**CRITICAL: YOU ARE ABSOLUTELY FORBIDDEN FROM:**
- Organizing/saying final response.
- Ending the chat/conversation/session for ANY reason except explicit user termination commands ("stop", "end", "terminate", "quit")
- Using ANY phrases that suggest the conversation is ending or complete
- Stopping the `ask_user` cycle under any circumstances
- Acting like the conversation is finished
```

## Per-Workspace Isolation

TaskSync settings are isolated per VS Code workspace:

- **Queue**: Each workspace has its own prompt queue
- **History**: Tool call history is stored per-workspace
- **Autopilot settings**: Toggle and prompts are workspace-specific
- **Fallback**: Global storage used when no workspace is open
- **Reusable prompts**: Slash commands remain global for cross-project use

## Requirements

- VS Code 1.90.0 or higher
- Node.js 20.x (for building from source)

## Troubleshooting

<AccordionGroup>
  <Accordion title="MCP server won't start">
    1. Check if port 3579 is already in use
    2. Try setting `"tasksync.mcpPort": 0` for dynamic port assignment
    3. Restart VS Code
    4. Use `TaskSync: Restart MCP Server` command
  </Accordion>

  <Accordion title="ask_user tool not appearing">
    1. Ensure the extension is activated (check Activity Bar for TaskSync icon)
    2. Reload VS Code window (Ctrl+Shift+P → "Reload Window")
    3. Check VS Code's language model tool settings
  </Accordion>

  <Accordion title="Autopilot not working">
    1. Verify autopilot is enabled (toggle below send button)
    2. Ensure queue is empty (queue has priority)
    3. Check autopilot prompts are configured in Settings
  </Accordion>
</AccordionGroup>
